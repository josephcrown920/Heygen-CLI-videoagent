---
name: HeyGen Studio Architecture
description: Monorepo structure, route mounting, and code-gen pipeline
---

**Monorepo layout:**
- `artifacts/api-server` — Express server, routes at `/api`, PORT from env
- `artifacts/heygen-studio` — Vite/React frontend, BASE_URL from import.meta.env
- `lib/api-spec/openapi.yaml` — single source of truth for HeyGen endpoints
- `lib/api-client-react` — TanStack Query hooks generated by Orval
- `lib/api-zod` — Zod schemas generated by Orval

**Route mounting:** `app.ts` → `router/index.ts` → health + heygen + fal routers

**Why:** Orval codegen keeps HeyGen types in sync. fal.ai routes are manually maintained (no codegen) since they're a free-form proxy.

**How to apply:** Add new HeyGen endpoints to openapi.yaml + run `pnpm run codegen` in lib/api-spec. Add new fal.ai models to `artifacts/heygen-studio/src/lib/models.ts` only.
